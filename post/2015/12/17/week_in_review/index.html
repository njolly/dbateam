This week in the DBA Universe - 2015-12-17

17 Dec, 2015 · by Nathan J · Read in about 3 min · (544 Words)
week_in_review dba

I was wrong about things getting more quiet leading up to Christmas. There’s been a lot of fascinating content in the three weeks since my last post.

SQL Server

  • Greg Low (SQLDownUnder) offers some video of presentations he did for Microsoft Ignite - the «things I wish developers knew about SQL Server» presentation looks particularly interesting
  • SQLskills has released all of the video content from their newsletters over the past year
  • Erik Darling (BrentOzar.com) shows how SSMS in SQL Server 2012 (and soon 2014) can now tell you how many rows a query has actually read. This could be very useful for query tuning
  • Kevin Farlee (Microsoft) highlights some major improvements for Always On in SQL Server 2016 - basic HA in standard edition and no more requirement for a single Windows domain!
  • On that note, the SQL Server Team has released SQL Server 2016 CTP 3.1. Most of the improvements seem focused on Business Intelligence / Data Analysis

MySQL

  • Percona have released Percona Server 5.6.27 and a bugfix for 5.5.46, and MariaDB have released MariaDB 5.5.47 as well. All of these are pretty much bug fix releases, including the upstream changes in MySQL 5.5.47
  • Percona have also released their first RC build of Percona Server 5.7. Based on the known bugs it’s clearly in its early stages, but it’s great to see a build so soon after the MySQL 5.7 GA
  • Simon Mudd (booking.com) demonstrates that upgrading your MySQL 5.6 instances to 5.7 might result in some unexpected side effects with expiring passwords
  • Vadim Tkachenko (Percona) reports some more Amazon Aurora benchmarks, but it’s worth reading just for some of the comments from an Aurora engineer describing a bit about how it works
  • Meanwhile, Manoj Kukreja (Pythian) has been benchmarking Google’s offering
  • Oracle have improved their MySQL Docker container release
  • If «injecting empty transactions» to fix GTID problems sounds like an unpleasant exercise, Daniel Guzmán Burgos (Percona) shows how you can do it more elegantly with a utility named mysqlslavetrx
  • Dmitri Kravtchuk (Oracle) has some InnoDB vs MyISAM benchmarks on 5.7. Thankfully, it’s nearly impossible to justify the use of MyISAM now
  • Daniël van Eeden (booking.com) has created a useful proof-of-concept MySQL plugin, using the new 5.7 Query Rewrite functionality, which standardises unicode characters
  • Yoav Abrahami (Wix) discusses the viability of MySQL as a NoSQL engine

General

  • Brent Ozar (BrentOzar.com) ponders how many servers a single DBA can comfortably manage. The deeper down you go, the less broadly you can work
  • If you bookmark one thing from this blog post, bookmark this: Linux Performance Analysis in 60 seconds. HighScalability mentions some great follow-up discussion on HackerNews
  • Twitter talks about how they perform reliability testing («we created a framework that can inject controlled failure conditions into the Twitter infrastructure»). You can create the finest test environment, but there’s just nothing quite like testing in production
  • Grant Fritchey (Red Gate) is learning R. As a DBA with a peripheral interest in data science, this blog series will be something to keep an eye on
  • There’s some retrospective analysis of a breach of Patreon’s data. tl;dr, a dev database was compromised and the data wasn’t masked. Oops
  • HighScalability points to an updated Red Book called «Readings in Database Systems». I am looking forward to reading these
