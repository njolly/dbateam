This week in the DBA Universe - 2015-11-25

25 Nov, 2015 · by Nathan J · Read in about 2 min · (372 Words)
week_in_review dba

I expect things will quiet down on the database news front as North American companies take some time off for Thanksgiving and head towards the Christmas holidays.

SQL Server

  • SQL Server 2012 SP3 is available! Kendra Little (BrentOzar.com) points out some great additions to DMVs which have been backported from SQL Server 2016
  • Brent Ozar had a neat list of things that are worth checking if your SQL Server suddenly becomes slow
  • The SQL Server DBCC SHOW_STATISTICS command is going to be even useful with this step-by-step guide on how to interpret the results

MySQL

  • Dimitri Kravtchuk (Oracle) has an interesting comparison of the 1M+ QPS metrics posted for both MySQL 5.7 and MariaDB 10.1. Possibly worth a read just for the comments discussion between Dimitri and Mark Callaghan
  • MyDumper 0.9.1 is available - this is quite a major release and includes a number of features that may make it a viable alternative to mysqldump for logical backups
  • If you’re looking to poke around MySQL 5.7 but don’t have the inclination to spin up your own VM / container, you can look into db4free.net, which now offers MySQL 5.7 databases free for testing
  • Vadim Tkachenko (Percona) offers more third-party analysis of Amazon’s Aurora service. There are a few interesting risks which I hadn’t considered
  • Shlomi Noach (booking.com) has a great, if slightly biased (he maintains an alternative product) run-down of limitations in MariaDB’s MaxScale product

General

  • ITNews reports on a December 1 cease-sale of Oracle SE and SE1 (in favour of SE2) which may have serious licensing implications for some users. If you use Oracle you might want to take a look
  • Robert Harris (StorageMojo) offers some analysis on Dell’s recent acquisition of EMC. This industry seems to be endlessly cyclical but even for on-premise compute the trend seems to be towards deploying on commodity baremetal rather than VMs on expensive storage, so I’m not sure how much mileage there is in EMC’s product suite
  • What affects remote workers the most? A small survey by Moishe Lettvin (Etsy) suggests it’s not discipline, but communication and loneliness that can be the biggest challenges. George Stocker (Jewelbots), another remote worker, offers some suggestions for trying to work around the biggest challenges of remote work
