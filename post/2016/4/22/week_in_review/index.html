This week in the DBA Universe - 2016-04-22

22 Apr, 2016 · by Nathan J · Read in about 2 min · (300 Words)
week_in_review dba

So, it’s been another month. I’ll have to find a title better than «this week in…».

Anyone interested in seeing updates for Oracle, DB2, PostgreSQL or Sybase ASE? I have to work with all of them now. Let me know!

SQL Server

  • I missed this by a day or two with my last blog, but I’m really excited to see that SQL Server Developer Edition is now free
  • Erik Darling (BrentOzar.com) offers advice on how to use Ola Hallengren’s great DBA scripts to update statistics without doing index maintenance. With modern storage systems you might find that index maintenance is costing you a lot more than it’s helping
  • SQL Server 2016 Release Candidate 3 is now available, and is being flagged as the «final» release candidate before official release
  • Microsoft has removed SQL CLR support from Azure SQL Database

MySQL

  • MariaDB 10.2 Alpha is now available. It’s create to see window functions even if they’re still quite buggy, though I’d rather have CTEs…
  • MySQL 5.7.12 is also now available, and introduces significant new features. It can act as a document store, offers a new API, an entirely new interactive shell with JS and Python support, and the introduction of a node.js connector alongside the updated Java and .NET connectors
  • Percona have bundled together their own monitoring scripts with Prometheus and Grafana to produce a beta of a new Percona Monitoring and Management tool
  • Percona XtraBackup 2.4.2 is generally available

General

  • Vadim Tkachenko and Peter Zaitsev (Percona) have done an exhaustive analysis on different compression algorithms (+Update - though the results don’t appear to have immediate practical implications
  • Diginomica offers a write-up on the promise of GPU-accelerated databases
  • Microsoft’s Insider program is already offering support for native Linux bash on Windows. Maybe I can have my cake and eat it too!
