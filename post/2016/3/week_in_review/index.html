This week in the DBA Universe - 2016-03-29

29 Mar, 2016 · by Nathan J · Read in about 2 min · (363 Words)
week_in_review dba

It’s tough to get back into the swing of things. New year, new job, new reasons to procrastinate. On account of being three months since my last update, I’ll stick to the highlights.

SQL Server

  • At an event it hosted earlier this month, Microsoft made the announcement that it is making SQL Server available on Linux in mid 2017, and that it’s already in «private preview» to interested customers. This could really help Microsoft attract existing Oracle customers, and it will definitely please a few DBAs whose environment is primarily running on Linux/Unix, but the real winners here are probably Microsoft. I’d wager they’d save a lot of memory getting Azure’s SQL Database engines running on Linux instead of Windows…
  • Brent Ozar offers his thoughts on the top 4 job tasks DBAs forget. I can attest to the value of all of these
  • SQL Server 2016 is already at Release Candidate 1. It seems likely we’ll see the full release in the next couple of months
  • BrentOzar.com has released new versions of their Blitz scripts: sp_Blitz, sp_BlitzIndex and sp_BlitzCache

MySQL

  • Vadim Tkachenko (Percona) has been doing some analysis of the CPU, Network and IO overheads of running MySQL in a docker container vs. baremetal. Results are promising: there’s minimal IO overhead and the CPU overhead also appears to be low, but the default port-forwarding docker configuration looks to pose some throughput problems
  • To accompany the release of Percona Server 5.7 (currently at 5.7.11), Percona have released new versions of XtraBackup (2.4.1) and Percona Toolkit (2.2.17) which finally support MySQL 5.7. Unless you have an extremely busy single-threaded workload (where 5.5 would probably suit you best), there probably aren’t any good reasons to avoid upgrading

General

  • Visual Studio Community Edition now offers support for the R language. R and other analytics/learning tools seem to be a significant part of Microsoft’s plans for SQL Server
  • In «Someties a variety of databases is THE database you need», Alex Slotnick (VividCortex) offers some analysis of a DZone survey on database maintenance, where he notes that nearly half of the respondents are running three, four or five different database technologies. And that’s just the ones they know about!
