This week in the DBA Universe - 2016-05-09

9 May, 2016 · by Nathan J · Read in about 1 min · (192 Words)
week_in_review dba

SQL Server

  • We have a release date for SQL Server 2016 - June 1. There’s a lot of new BI/Analytics related features, but Brent Ozar has a good summary of the highlights that most operational DBAs would care about
  • Ayo Olubeko (Microsoft) blogs about improvements in the SQL Server tools coming as part of SQL Server 2016. I think the nicest improvement here is centralising the downloads, which have been a nightmare to navigate in the past couple of releases

MySQL

  • Preetam Jinka (VividCortex) discusses the benefits that come from being able to index virtual columns, available since MySQL 5.7.8
  • Peter Zaitsev (Percona) shares Best Practices for Configuring Optimal MySQL Memory Usage. There’s no surprises here, just good, practical advice
  • In the same vein, Vadim Tkachenko (Percona) looks at how CPU governor performance affects MySQL
  • MariaDB has a series of blogs about an upcoming ColumnStore feature - an adaptation of InfiniDB - targeted for beta availability later this month

General

  • The title is a bit misleading, but ArsTechnica’s «Power tools: Sorting through the crowded specialized database toolbox» provides some useful notes about how a few companies chose their Key Value Store databases
